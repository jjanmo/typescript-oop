Typescript Setup

tsc -w : ts파일이 변경되는 것을 지속적으로 tracking하여 자동적으로 JS파일로 컴파일해주는 명령어

tsconfig compile option

📌 자세한 사항은 여기를 참고

  • "incremental": true => 기존에 컴파일한 것과 비교하여 수정된 부분만 다시 컴파일하는 옵션, 기존의 컴파일한 정보를 저장하기때문에 PC의 용량이 늘어날 수 있다.
  • "target": "es5" => 어떤 버전으로 컴파일할지는 설정하는 옵션
  • "module" : "commonjs" => 컴파일할 때, 모듈시스템을 어떻게 구현할지에 대한 설정
  • "lib": [] => 어떤 라이브러리를 이용할지에 대한 설정, 보통, 따로 지정하지 않고, target 옵션에서 따라나오는 이용한다.
  • "allowJs": true => 타입스크립트 파일에서 자바스크립트를 컴파일하는 것을 허용
  • "checkJs": true => (컴파일한) 자바스크립트에서 잘못된 것이 있다면 에러가 뜰수 있도록 허용하는 옵션
  • "declaration": true / "declarationMap": true => 타입정의에 관련된 것으로서 내가 만든 코드를 라이브러리 형식으로 다른 사람에게 제공하는 경우에만 사용한다. 일반적인 경우 설정하지않는다.
  • "sourceMap": true => 디버깅할 때 유용하게 사용할 수 있는 옵션
  • "outFile": "./" => 컴파일한 자바스크립트 파일을 딱 하나로 만들때 사용하는 옵션
  • "composite": true => incremental 옵션과 같이 사용하는 것으로서, 이전에 빌드된 내용을 기억하여 다음에 빌드할때 더 빠르게 빌드할 수 있도록 도와준다.
  • "tsBuildInfoFile": "./" => "incremental": true 로 설정할 때, 이와 관련된 정보를 넣는 파일의 위치를 지정하는 옵션
  • "removeComments": true => 컴파일시 주석을 모두 삭제
  • "noEmit": true => 컴파일 에러체크만 하고 자바스크립트로 변환하지는 않는다.
  • "isolatedModules": true => 각각의 파일을 다른 모듈로 만들고자 할 때, 사용하는 옵션
  • "strict": true => 일반적으로 true로 놓고 사용하는 것이 좋다. false로 놓으면 일반 자바스크립트와 같기때문에 굳이 바꿀필요 없다. 만약에 false로 바꾼다면, 아래에 구체적인 type-checking옵션을 설정해야한다.
  • "outDir": "./build" => 컴파일한 자바스크립트의 위치
  • "rootDir": "./src" => 어디서부터 ts를 컴파일할 것인지, ts 파일의 시작점

ETC

  • "exclude": [] => 컴파일에서 제외할 파일 / 디렉토리 설정
  • "include": [] => 컴파일할 파일 / 디렉토리만 설정
      {
        "compileOptions": {},
        "include": [],
        "exclude": []
